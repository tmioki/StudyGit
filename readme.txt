<git 調査>
svn サーバへのコミットを減らすためには有効かも。trunkを安定させることができそう。
実質コードレビューを行うにはgitかsvn の mod ブランチを活用するしか無いのでは。
  
冗談のように面倒くさい（少なくともローカルリポジトリへコミット、リモートリポジトリへのコミットの２段階必要）
リモートリポジトリをどういう単位で作成するか十分検討が必要
（コンポーネントを横断する機能においてはリモートリポジトリがsvnサーバと同じになりかねない）

eclipse と git の連携でやりたいこと。
1. ローカルリポジトリへの push / pull / history / commit / add: 確認
2. リモートリポジトリへの push / pull : 確認
2-1. svn リポジトリからの pull : 未確認
2-2. svn リポジトリへの push : 未確認

<git の設定手順>
i)eclipse
egit というプラグインを使う。
インストール方法は他のプラグインと同じ。（2012/12/23現在、正式リリース版ではないので注意）
githubを使う場合はRSAの設定をちゃんとすること。Pushできなくなります。
（コマンドラインでの git push origin に失敗するのと同じ原因）
 
eclipse での RSA 設定は、
Eclipse->環境設定->NetworkConnections->SSH2からGeneralタブの private key に使う鍵が設定されている事を確認する。
それ以外はググれ。
